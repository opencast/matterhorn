This is the documentation for the REST endpoint for the ingest service. There are two ways how media can be ingested:

Thin Client: A simple POST form ingests a new media package. Input types of form fields must conform to Opencast metadata.

Thick Client: Client must collect all the media, catalogs and attachments, create a MediaPackage as a manifest.xml and submit everything as a zipped file.
